# Implementation Plan: Migrate from pre-commit to prek

**Branch**: `feature/migrate-to-prek` | **Date**: 2026-05-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/007-migrate-to-prek/spec.md`

## Summary

Replace the `pre-commit` Python tool with `prek` (v0.4.3, Rust-native) as the sole git hook runner
in this project. CI already uses `prek`; this plan completes the migration by removing the `pre-commit`
and `pre-commit-update` packages, converting the hook configuration to native TOML format, and
eliminating every developer-facing reference to the `pre-commit` tool. The skill
`pre-commit-hooks-create` is renamed to `git-hooks-create` to remove the final naming trace.

**Approach**: All three priority tiers are included. No backward compatibility with `pre-commit` is
maintained after completion.

## Technical Context

**Language/Version**: Python 3.14 (project); Rust binary (prek — no Python dep at runtime)

**Primary Dependencies**:
- `prek>=0.4.3` (hook runner — replaces `pre-commit` and `pre-commit-update`)
- `uv` (package management and venv execution)

**Storage**: N/A (tooling migration, no persistent data)

**Testing**: Manual validation — `uv run prek run --all-files` must pass after each tier

**Target Platform**: Windows (dev), Linux (CI — already passing)

**Project Type**: Tooling migration / documentation update

**Performance Goals**: N/A — prek is already faster than pre-commit

**Constraints**: Zero functional regression in hook behaviour; all existing hooks must continue to block the same violations

**Scale/Scope**: 17 hooks in config, 13 hook scripts, ~12 files updated across skills/docs/config

### P3 — Config File Migration (depends on P2 complete)

**Goal**: Replace `.pre-commit-config.yaml` with `prek.toml`; eliminate all `".pre-commit-config.yaml"` references.

#### Task P3.1 — Convert YAML config to TOML

```powershell
uv run prek util yaml-to-toml --output prek.toml
```

Review the generated `prek.toml`. Confirm all hooks present. Re-add any operational comments
(YAML comments are not preserved by the converter).

Key things to verify in the TOML output:
- All `priority` values preserved (prek-only extension — already supported)
- `always_run: true` / `pass_filenames: false` hook options correct
- `additional_dependencies` block for `codespell` preserved
- `ci.autofix_prs` top-level key — prek tolerates this; leave it or remove it (it has no effect without pre-commit-ci)

#### Task P3.2 — Smoke test with TOML config

```powershell
uv run prek run --all-files
```

prek prefers `prek.toml` over `.pre-commit-config.yaml` when both exist. Confirm all hooks pass.

#### Task P3.3 — Delete `.pre-commit-config.yaml`
