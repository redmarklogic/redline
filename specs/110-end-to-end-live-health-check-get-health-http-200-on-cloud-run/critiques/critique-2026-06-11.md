## Engineering Lens Findings

### Architecture Soundness

| ID | Severity | Finding | Suggestion |
|----|----------|---------|------------|
| E1 | RECOMMENDATION | Manual-apply image regression: after CI has deployed newer SHAs, any later reviewed `terraform apply` requires `-var image_tag=<sha>` — an operator who passes a stale SHA silently rolls the serving image backwards. The drift note says no stale tag re-applies "silently", but the operator choosing the tag is the silent failure point. | Add to quickstart.md drift note: before any manual apply, read the currently-serving image (`gcloud run services describe staging-redline-api --region australia-southeast1 --format 'value(spec.template.spec.containers[0].image)'`) and pass that SHA unless deliberately rolling back. |

### Failure Mode Analysis

| ID | Severity | Finding | Suggestion |
|----|----------|---------|------------|
| E2 | RECOMMENDATION | The CI health-check retry window (~60 s) may be tight on a scale-to-zero service: cold start = instance scheduling + image pull + 10 s probe delay; a large image can push past 60 s and fail a healthy deploy. | Either extend the retry window to ~120 s, or (better) gate the curl on service readiness first: poll `gcloud run services describe ... --format 'value(status.conditions[0].status)'` until Ready, then curl. Update T023 wording accordingly. |
