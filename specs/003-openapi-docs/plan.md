# Implementation Plan: OpenAPI Documentation

**Branch**: `feature/003-openapi-docs` | **Date**: 2026-06-09 | **Spec**: [spec.md](./spec.md)

**Status**: Draft

## Summary

`create_app()` in `src/marker/api/main.py` currently passes `docs_url=None, redoc_url=None` to FastAPI, suppressing all interactive documentation. This feature re-enables the built-in Swagger UI (`/docs`) and OpenAPI schema (`/openapi.json`), adds human-readable app metadata (title, version), and updates `tasks/run-app.ps1` to open the browser at `/docs` instead of the root URL. No new source files; no new dependencies; no layer changes.

## Design Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Where metadata lives | Inline kwargs on `FastAPI(...)` in `create_app()` | Single construction point; no separate config object warranted for three string fields. |
| D2 | `/redoc` | Enabled (FastAPI default) | No cost; bonus value; not a stated requirement either way. |
| D3 | Auth on `/docs` | No auth | Developer-only tool; no external exposure. Consistent with the assumption in spec.md. |
| D4 | `run-app.ps1` URL | Change `$Url` used in `Start-Process` to `"$Url/docs"`; readiness probe stays on `$Url` (root) | Root URL probe is more reliable (no auth redirect); docs URL is what the developer needs in the browser. |

## Phased Delivery

### Phase 0: Re-enable docs + app metadata

**Goal**: `GET /docs` returns 200 and the Swagger UI HTML; `GET /openapi.json` returns a valid schema listing POST /skeletons.

**Changes**:

1. `src/marker/api/main.py` — update `create_app()`:
   - Remove `docs_url=None, redoc_url=None`.
   - Add `title="Marker API"`, `version="0.1.0"`.

2. Check `tests/marker/api/test_openapi.py` — confirm it still passes after docs are re-enabled (it may have been written to call `app.openapi()` directly, which works regardless; or it may probe `/openapi.json` via TestClient, which requires docs enabled).

**Acceptance Gate**:

- [ ] `GET /docs` returns `200` via `TestClient`.
- [ ] `GET /openapi.json` returns a valid OpenAPI 3.x schema containing `POST /skeletons`.
- [ ] `python -m pytest tests/marker/api -v` green.

---

### Phase 1: Update run-app.ps1 to open /docs

**Goal**: Running `tasks/run-app.ps1` opens the browser directly to the docs page.

**Changes**:

1. `tasks/run-app.ps1` — in the `Start-Process $Url` call, change target from `$Url` to `"$Url/docs"`.

**Acceptance Gate** (manual):

- [ ] Run `tasks/run-app.ps1`; confirm browser opens to `http://127.0.0.1:8765/docs`.
- [ ] Swagger UI renders with "Marker API 0.1.0" header and POST /skeletons listed.
