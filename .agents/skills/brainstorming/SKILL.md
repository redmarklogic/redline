## Common Mistakes

| Mistake | Fix |
|---|---|
| Skipping brainstorming and jumping straight to implementation | Always run brainstorming before creative work; implementation without intent exploration produces the wrong thing |
| Treating the first idea as the only option | Generate at least three alternatives before evaluating; anchoring on the first option forecloses better solutions |
| Confusing brainstorming output with a spec | Brainstorming produces intent and constraints; hand off to spec-kit for a formal spec before engineering begins |
