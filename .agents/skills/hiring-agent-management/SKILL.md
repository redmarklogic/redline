## Common Mistakes

| Mistake | Fix |
|---|---|
| Writing a JD before identifying the skill gap | Map the gap to an existing or new skill first; the JD is derived from skills, not written in isolation |
| Assigning the same skill to multiple agents as their primary domain | Each skill has one owning agent tier; overlap creates conflicting advice |
| Skipping the Agent Topology Sync when adding a new agent | Run the sync ceremony before finalising any new hire; topology changes affect routing for all agents |
